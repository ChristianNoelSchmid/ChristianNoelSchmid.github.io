Software Focuses

In software, there are several practices that are important to every project. These are the focuses I have when I develop:

  1. Maintainability - software must be understandable to other developers, and capable of being built upon. The demands of the modern world require upgrades and improvements to software. I always emphasize readability and future considerations of my code.
  2. Security - software must be written in a way that can be received, and can respond to users without affecting the integrity of the system itself. I've worked to research the best practices for security in modern software development.
  3. Performance - software should be written in a way that maximizes the use of the hardware running it. My training has emphasized proper usage of memory and CPU to ensure the maximum potential for a program. On top of this, parallel programming is the present, and future of the software industry. I've striven to implement code that performs well in a multi-threaded environment.
